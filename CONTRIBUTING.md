# Contributing

First of all, thank you for your interest in contributing to the **IoT Based Heart Rate Monitoring System** project.

Contributions from the community help improve the quality, reliability, and functionality of this project. Whether you are fixing bugs, improving documentation, optimizing code, or adding new features, your support is greatly appreciated.

---

## Ways to Contribute

You can contribute in several ways, including:

- Reporting bugs
- Fixing existing issues
- Improving documentation
- Enhancing code readability
- Optimizing program performance
- Adding new IoT features
- Supporting additional sensors
- Improving cloud integration
- Creating example projects
- Updating library compatibility

---

## Before You Start

Before making any changes, please:

- Read the project documentation carefully.
- Check existing Issues to avoid duplicate work.
- Open a new Issue if you plan to implement a major feature.
- Keep changes focused and well documented.

---

## Development Guidelines

Please follow these guidelines while contributing:

- Maintain the existing coding style.
- Write clean and readable code.
- Add comments wherever necessary.
- Do not introduce unnecessary libraries.
- Preserve compatibility with the NodeMCU ESP8266 platform.
- Test all modifications before submitting.
- Keep commits small and meaningful.

---

## Pull Request Process

1. Fork this repository.
2. Create a new feature branch.

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes.

4. Commit your changes.

```bash
git commit -m "Add: Short description of your changes"
```

5. Push the branch.

```bash
git push origin feature/your-feature-name
```

6. Open a Pull Request.

---

## Commit Message Style

Examples of good commit messages:

```text
Add: OLED startup animation

Fix: ThingSpeak upload issue

Improve: Heart rate calculation

Update: README documentation

Refactor: Wi-Fi initialization
```
