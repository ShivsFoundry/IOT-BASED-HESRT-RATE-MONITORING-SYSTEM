# Security Policy

## Supported Versions

The latest version available in the main branch is actively maintained.

| Version | Supported |
|:--------:|:---------:|
| Latest Release | ✅ |
| Older Versions | ❌ |

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

Please **do not disclose security vulnerabilities publicly** until they have been reviewed and addressed.

---

## How to Report

Please include the following information:

- Description of the vulnerability
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Hardware used
- Arduino IDE version
- ESP8266 Board Package version
- Library versions
- Serial Monitor logs (if available)

---

## Response Process

After receiving a vulnerability report, the following process will be followed:

1. Acknowledge the report.
2. Verify the reported issue.
3. Investigate the root cause.
4. Develop and test a fix.
5. Release an updated version if necessary.
6. Credit the reporter where appropriate.

---

## Scope

This security policy applies to:

- Firmware
- Documentation
- Build configuration
- Repository contents

It does not apply to:

- Third-party libraries
- Arduino IDE
- ESP8266 Core
- ThingSpeak platform
- External hardware

---

## Best Practices

When using this project, it is recommended to:

- Never publish your Wi-Fi credentials.
- Never expose your ThingSpeak API keys.
- Use private repositories for sensitive deployments.
- Keep Arduino libraries updated.
- Keep the ESP8266 board package updated.
- Regularly review cloud platform permissions.

---

## Disclaimer

This project is intended primarily for educational, research, and learning purposes.

It is **not** designed, certified, or approved for clinical, diagnostic, or life-critical medical applications.

Users are responsible for validating the system before deploying it in any real-world healthcare environment.

---

## Contact

If you believe you have found a security issue, please open a **private security report** if GitHub Security Advisories are enabled for this repository. Otherwise, contact the repository maintainer through GitHub before making any public disclosure.
